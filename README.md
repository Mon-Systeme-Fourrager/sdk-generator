# sdk-generator

The generator used to generate the external SDKs for MSF.

To install this, either:
- run `pip install git+https://github.com/Mon-Systeme-Fourrager/sdk-generator.git`
- add `git+https://github.com/Mon-Systeme-Fourrager/sdk-generator.git` to your requirements file

To publish a new version of the generator, update the version number in `pyproject.toml`, then run the following commands:
- `python -m build`   (Note: You might need to install build if not already done: `pip install build`)
- `twine check dist/*`    (Note: This is just to make sure the build has nothing missing. You might need to install as well: `pip install twine`)
- `git tag {version number} -m "{Release Message}"`
- `git push origin {version number}`

## SDK Operations

The following sections describe the common operations for the SDKs generated by this tool.

### Installing an SDK in a project

To use an SDK in a project, either:
- run `pip install git+https://github.com/Mon-Systeme-Fourrager/{sdk-name}`
- add `git+https://github.com/Mon-Systeme-Fourrager/{sdk-name}` to your requirements file

### Regenerating SDK code

Prerequisites: install sdk-generator (see top of this README).

From within the target SDK's directory, run the generator with its OpenAPI spec and constants template:

```bash
sdk-generator <openapi_spec.yaml> <output_package> --constants-template-path constants.jinja2
```

Refer to each SDK's README for the exact command and any additional setup steps.

### Publishing a new SDK version

1. Update the version number in `pyproject.toml`
2. Regenerate the SDK code (see above)
3. Build the source distribution and wheel: `python -m build`   (Note: You might need to install build if not already done: `pip install build`)
4. Validate the output: `twine check dist/*`   (Note: You might need to install twine as well: `pip install twine`)
5. Create a tag: `git tag {version number} -m "{Release Message}"`
6. Push the tag: `git push origin {version number}`
